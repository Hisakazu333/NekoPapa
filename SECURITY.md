# 安全策略

## 支持范围

NekoPapa 当前处于开发预览阶段，尚无稳定公开发行通道。安全修复以最新
`main` 为准；旧提交、开发安装包和未合并分支不承诺单独维护。

## 报告漏洞

不要为未修复漏洞创建公开 Issue，也不要在截图、日志或 PR 中提交令牌、私钥、
本机绝对路径或可直接利用的细节。

优先通过仓库的 GitHub Security Advisory 私密报告功能联系维护者。若该入口尚未
启用，请通过仓库所有者 GitHub 主页提供的私密联系方式发送最小摘要，并等待安全
沟通渠道，不要公开披露利用步骤。

报告请包含：

- 受影响提交、平台、架构和版本；
- 影响范围和攻击前提；
- 最小复现或概念验证；
- 预期与实际的权限边界；
- 已采取的临时缓解方式；
- 是否涉及第三方 SDK、Live2D 资产或上游漏洞编号。

维护者会先确认接收，再评估严重性、修复范围和披露时间。开发预览阶段不承诺固定
SLA，但会尽量避免在修复可用前公开利用细节。

## 重点安全边界

- Tauri WebView 不能传入任意 shell 命令、可执行路径或未校验参数。
- Native Stage 只作为受监督的子进程运行，不开放本地网络控制端口。
- Stage 协议必须拒绝未知版本、未知消息和畸形 JSON；协议输出与日志分离。
- 访问令牌、账户数据和私有模型路径不得进入协议、日志、Issue 或构建产物。
- Content Security Policy、Tauri capabilities 和 `macOSPrivateApi` 的使用必须在发布
  前单独审计。开发配置不能自动继承为发布配置。
- 下载、更新、插件和模型导入必须校验来源、完整性和许可，不能执行资源包中的
  脚本。
- Stage 崩溃或恶意输出不能阻止主窗口启动，也不能造成无限重启。

## 依赖与供应链

Dependabot 覆盖 npm、Cargo 和 GitHub Actions。CMake `FetchContent` 依赖必须固定
版本或提交。引入新的 Action、SDK、二进制和模型时，PR 必须记录来源、许可证、
校验方式和打包影响。

Live2D Cubism SDK、Cubism Core 和示例模型受其各自许可约束，不因仓库自有代码
使用 Apache-2.0 而改变。
